Background

Mini2440 comes with SuperVivi bootloader preinstalled. Actually I prefer to use u-boot bootloader because it's flexibility. The official u-boot website is at http://www.denx.de/wiki/U-Boot/WebHome.

NOTE: u-boot actually is under a massive internal reorganization that invalidate actual setup for ARM based systems; that's prevents u-boot to successfully build for ARM; the community is under pressure to rebuild all the setup for ARM; this - of course - impact also in Mini2440 porting under u-boot; for this reason we will use the pre-built image of u-boot supplied on Mini2440 website download page site or on original DVD.

Needed tools

Hardware

  • FriendlyARM Mini2440 board;
  • RS232 straight cable (Mini2440 RS232 is configured as DCE - cable provided by vendor);
  • USB type A to type B cable (provided by vendor);
  • 5Vdc power supply (vendor provides the power supply with US plug);

Software

  • u-boot prebuilt version 2010-07-01 (checkout at http://www.friendlyarm.net/downloads);
  • s3c2410_boot_usb software (provided on the zip file downloaded on FriendlyARM website - see below);
  • Minicom or equivalent (can be installed from your repo);

Preliminary

Check the NAND memory on your board

Before loading uboot you have to check the size of the NAND memory to decide which version of the pre-built images has to be load.

As you can see in the picture above, the actual NAND mounted on the Mini2440 is a 256Mb; check on the NAND manufacturer's website the actual size of the NAND mounted on your board.

Preparing u-boot and his tool

  • Open a new shell on host pc, unzip the downloaded u-boot: 
$ unzip u-boot_20100701.zip -d u-boot_20100701
Archive:  u-boot_20100701.zip
   creating: u-boot_20100701/tools/
  inflating: u-boot_20100701/tools/s3c2410_boot_usb  
  inflating: u-boot_20100701/tools/s3c2410_boot_usb_old  
  inflating: u-boot_20100701/tools/usbpush  
  inflating: u-boot_20100701/u-boot-128M.bin  
  inflating: u-boot_20100701/u-boot-256M.bin  
  inflating: u-boot_20100701/u-boot-v132.bin  
$

As shown above with the archive comes three version of u-boot, two of them padded for different NAND sizes; on our case we have to use the image padded for 256Mb memory (file u-boot-256M.bin).

  • Check the size (in bytes) of proper u-boot image we are about to load; we have installed a 256Mb flash, so we are interested in the size of that image: 
$ cd u-boot_20100701/
~/qemuarm/u-boot_20100701$ ls -l
totale 728
drwxr-xr-x 2 fpiraneo fpiraneo   4096 2010-12-01 19:22 tools
-rw-r--r-- 1 fpiraneo fpiraneo 242332 2009-11-06 10:12 u-boot-128M.bin
-rw-r--r-- 1 fpiraneo fpiraneo 242360 2010-07-01 15:54 u-boot-256M.bin
-rw-r--r-- 1 fpiraneo fpiraneo 246164 2010-05-25 19:16 u-boot-v132.bin
~/qemuarm/u-boot_20100701$

Actual size is 242360 bytes; please remember this data;

  • With u-boot comes also the utility usbpush; this utility can be used to upload u-boot image into flash with supervivi bootloader. Looking into the u-boot_20100701/tools/ check that the usbpush file is executable; make it otherwise: 
$ cd u-boot_20100701/tools/
~/qemuarm/u-boot_20100701/tools$ ls -l
totale 44
-rw-r--r-- 1 fpiraneo fpiraneo 12170 2010-11-27 18:10 s3c2410_boot_usb
-rw-r--r-- 1 fpiraneo fpiraneo 14086 2009-05-23 09:46 s3c2410_boot_usb_old
-rw-r--r-- 1 fpiraneo fpiraneo 13246 2009-11-10 19:31 usbpush
~/qemuarm/u-boot_20100701/tools$ chmod a+x usbpush
~/qemuarm/u-boot_20100701/tools$ ls -l
totale 44
-rw-r--r-- 1 fpiraneo fpiraneo 12170 2010-11-27 18:10 s3c2410_boot_usb
-rw-r--r-- 1 fpiraneo fpiraneo 14086 2009-05-23 09:46 s3c2410_boot_usb_old
-rwxr-xr-x 1 fpiraneo fpiraneo 13246 2009-11-10 19:31 usbpush
~/qemuarm/u-boot_20100701/tools$

Install

Loading u-boot into Mini2440 memory

Refers to the following picture for all operations.

  • Check that the power switch S1 switch is in OFF position; the power led near the switch will be OFF and the switch in the RIGHT position (near the display);
  • As pictured below connect USB, RS232 and power to your board; obviously connect the other side of the cables to the host PC; if you try to connect the power adapter of the Mini2440 to the host PC, you are not qualified to perform the following procedure, so stop now!
  • Switch the boot memory selector S2 switch to NOR position (LEFT position);
  • On host computer, start minicom;
  • Setup minicom with the following communication parameters: 115200 8N1;
  • Powerup your board with the S1 switch and on minicom you'll see the following: 
##### FriendlyARM BIOS 2.0 for 2440 #####
[x] format NAND FLASH for Linux
[v] Download vivi 
[k] Download linux kernel 
[y] Download root_yaffs image 
[a] Absolute User Application
[n] Download Nboot for WinCE 
[l] Download WinCE boot-logo
[w] Download WinCE NK.bin 
[d] Download & Run 
[z] Download zImage into RAM 
[g] Boot linux from RAM 
[f] Format the nand flash 
[b] Boot the system 
[s] Set the boot parameters 
[u] Backup NAND Flash to HOST through USB(upload) 
[r] Restore NAND Flash from HOST through USB 
[q] Goto shell of vivi 
[i] Version: 0945-2K
Enter your selection:

NOTE: Actually on the NOR memory we leave the Vivi bootloader that we can use to handle the content of the NAND memory; this to avoid the use of a JTAG interface;

  • Recall the vivi shell with q command and type the following: 
Supervivi> load ram 0x32000000 242360 u-boot
USB host is connected. Waiting a download.

(Here I divide the hex digits to easy the reading: 0x3200 0000)

  • On the host PC, open another console and start the download of u-boot image: 
$ sudo ./usbpush ../u-boot-256M.bin 
csum = 0x3a3e
send_file: addr = 0x30000000, len = 0x0003b2b8
$
  • After issuing the command usbpush the supervivi reacts: 
Now, Downloading [ADDRESS:32000000h,TOTAL:242370]
RECEIVED FILE SIZE:  242370 (118KB/S, 2S)
Downloaded file at 0x32000000, size = 242360 bytes
Supervivi>

This confirm that u-boot has been received by your Mini2440;

  • Now it's time to start the downloaded u-boot; on minicom issue the following: 
Supervivi> go 0x32000000
go to 0x32000000
  argument 0 = 0x00000000
  argument 1 = 0x00000000
  argument 2 = 0x00000000
  argument 3 = 0x00000000

U-Boot 1.3.2-mini2440 (Feb 24 2010 - 13:04:49)

I2C:   ready
DRAM:  64 MB
Flash:  2 MB
NAND:  256 MiB
Found Environment offset in OOB..
USB:   S3C2410 USB Deviced
In:    serial
Out:   serial
Err:   serial
MAC: 08:08:11:18:12:27
Hit any key to stop autoboot:  0 
MINI2440 #

Well, now u-boot has started, so stroke a key before the 3 seconds delay expires to get the prompt;

Configuring and saving u-boot into flash

  • The NAND has to be prepared with the scrub command; this actually delete the bad block table of the memory re-issuing a new one; it's safe to rebuild the bad block table without keeping the manufaturer's one: 
MINI2440# nand scrub
NAND scrub: device 0 whole chip
Warning: scrub option will erase all factory set bad blocks!
         There is no reliable way to recover them.
         Use this command only for testing purposes if you
         are sure of what you are doing!

Really scrub this NAND flash? <y/N> type "Y" then "Enter" here!
Erasing at 0x7a0000 --   3% complete.
NAND 256MiB 3,3V 8-bit: MTD Erase failure: -5
Erasing at 0xe8e0000 --  91% complete.
NAND 256MiB 3,3V 8-bit: MTD Erase failure: -5
Erasing at 0xffe0000 -- 100% complete.
Bad block table not found for chip 0
Bad block table not found for chip 0
OK
MINI2440 #

NOTE: Remember to confirm the action when required!

  • Next action is to create the bad block table issuing the following command: 
MINI2440 # nand createbbt
Create BBT and erase everything ? <y/N> type "Y" then "Enter" here!
Skipping bad block at  0x00940000                                            
Skipping bad block at  0x0eb60000                                            
Skipping bad block at  0x0ff80000                                            
Skipping bad block at  0x0ffa0000                                            
Skipping bad block at  0x0ffc0000                                            
Skipping bad block at  0x0ffe0000                                            

Creating BBT. Please wait ...
MINI2440 #

Note that the command should almost quick to complete but it's very likely to fail, so it has to be restarted!

Don't forget to confirm the action when required!

  • Now save the actual u-boot (just loaded with supervivi) into flash: 
MINI2440# nand write.e 0x32000000 0x0 0x0003b2b8

NAND write: device 0 offset 0x0, size 0x3b2b8

Writing data at 0x3b000 -- 100% complete.
 242360 bytes written: OK
MINI2440 #

NOTE: The last hex digit is the size of loaded u-boot in hex! You can pick this data on the answer of the usbpush command as above!

  • Partition the NAND: 
MINI2440# dynpart
mtdparts mtdparts=mini2440-nand:0x00040000(u-boot),0x00020000(u-boot_env),0x00500000(kernel),0x0faa0000(rootfs)
MINI2440 #
  • Create the dynamic environment space for u-boot and save it: 
MINI2440# dynenv set u-boot_env
device 0 offset 0x40000, size 0x20000
45 4e 56 30 - 00 00 04 00
MINI2440 # saveenv
Saving Environment to NAND...
Erasing Nand...Writing to Nand... done
MINI2440 #

Final notes

  • Poweroff your Mini2440;
  • Move the S2 boot switch to NAND position;
  • Powerup again the Mini2440 and check minicom: 
U-Boot 1.3.2-mini2440 (Feb 24 2010 - 13:04:49)

I2C:   ready
DRAM:  64 MB
Flash:  2 MB
NAND:  256 MiB
Found Environment offset in OOB..
USB:   S3C2410 USB Deviced
In:    serial
Out:   serial
Err:   serial
MAC: 08:08:11:18:12:27
Hit any key to stop autoboot:  0 

Loading from NAND 256MiB 3,3V 8-bit, offset 0x60000
** Unknown image type
Wrong Image Format for bootm command
ERROR: can't get kernel image!
MINI2440 #

Well, this means that u-boot is looking for a kernel and is not able to find it. Refers to my next note about the creation and the installation of a Mini2440 kernel.
