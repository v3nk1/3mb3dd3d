Needed System

  • I used the finished buildroot system (tutorial can be found here)
  • Together with an NFS server for the filesystem (tutorial can be found here)

Buildroot configuration

We will now make the necessary configurations to build QT for the OK6410 development board. First of all move into your buildroot directory and start the menuconfig: 

$ make menuconfig

And make the following changes 

Package Selection for the target --->
       Graphic libraries and applications (graphic/text) --->
            [*] Qt --->
                   [*] Compile and install demos and examples (with code)
                   [*] Approve free license

This is the basic system we will start from, but you can always make some personal changes. Exit the menuconfig and compile the new system: 

$ make

This can take some time and after the compilation has finished, inside the output/images directory you can find a freshly created rootfs.tar filesystem. The NFS tutorial explained already how to use this file to mount it as NFS root filesystem: 

$ sudo rm -rf ~/nfs_root/*
$ sudo cp ~/buildroot-2011.05/output/images rootfs.tar ~/nfs_root/
$ cd ~/nfs_root
$ sudo tar -xf rootfs.tar

Open a minicom terminal and reboot the board. It will mount our filesystem containing QT.

Accessing the QT tutorials

Now move to the minicom terminal, login (using root). The QT tutorials can be found inside the /usr/share/qt folder. So: 

# cd /usr/share/qt

Inside this directory we can find a demo and an examples folder where the included demos and examples are stored. Let's for example go into the widgets/sliders example directory: 

# cd /usr/share/qt/examples/widgets/sliders

You can try the sliders example by typing: 

# ./sliders -qws

You will see a window appearing on the screen containing several slider widgets. Unfortunately we can not do anything because there's no mouse or touchscreen configured. That's what we will do now! Press CTRL+C to exit the example.

Configuring the input interfaces

Two methods will be discussed here: the touchscreen and a USB mouse. We will configure QT to handle those input devices and configure the Linux kernel to calibrate the touchscreen.

Using the touchscreen

First of all we need to make the necessary changes to the Buildroot QT configuration to access the touchscreen. Open a new terminal and open the configuration of buildroot: 

$ cd ~/buildroot-2011.05
$ make menuconfig

Make the changes for the touchscreen: 

[*] Package Selection for the target  --->
          [*] Graphic libraries and applications (graphic/text)  --->
                 [*]   Qt  --->
                     Mouse drivers  --->
                           [*] tslib

Exit buildroot and make: 

$ make

Replace the NFS filesystem with the fresh one (see above). Switch to the minicom terminal, reboot the board and login into Builroot. First thing to do is to create the necessary environment variables: 

# export TSLIB_TSDEVICE=/dev/input/event1
# export TSLIB_TSEVENTTYPE=INPUT
# export TSLIB_CONFFILE=/etc/ts.conf
# export TSLIB_CALIBFILE=/etc/pointercal

After the configuration we can do the calibration of the touchscreen to make it work proparly: 

# ts_calibrate

And follow the instructions on the screen. After calibrating you can start testing it using the following command: 

# ts_test

Let's try the QT example again using the touchscreen. But first QT needs to know which device to listen to: 

# export QWS_MOUSE_PROTO="Tslib:/dev/input/event1"

Then we can run the example: 

# cd /usr/share/qt/examples/widgets/sliders
# ./sliders -qws

The -qws parameter is necessary for our embedded system because we don't run a QWS server. The -qws command creates a QWS server only for this app. You will discover that this example is not really made for the touchscreen because all the buttons are very small, but in the end it works! ;-)

Using a universal USB mouse

The second option is to plug in a USB mouse into the USB port of the OK6410 board and use this mouse as a pointer device. So let's edit the buildroot configuration again: 

$ cd ~/buildroot-2011.05
$ make menuconfig

 [*] Package Selection for the target  --->
          [*] Graphic libraries and applications (graphic/text)  --->
                 [*]   Qt  --->
                     Mouse drivers  --->
                           [*] pc
                           [*] tslib (you can leave this one active if you want to use the touchscreen some more)

$ make

Replace the old filesystem with the new one from the buildroot-2011.05/output/image/rootfs.tar (as explained above), reboot the board and go into the minicom terminal and plugin the USB mouse. First we have to discover which /dev/input/mouseXX device is connected to your USB mouse (replace the XX by a number going from 0 to 3, you can check this inside the following folder on the board: /dev/input/) 

# cat /dev/input/mouseXX | hexdump

On one device you will see a lot of data appearing when you move the mouse. This means that you have found the right mouseXX device. (also use CTRL+C to end the cat command). For me it was the mouse1 input device, so replace mouse1 in the next command with your mouse device: 

# export QWS_MOUSE_PROTO="MouseMan:/dev/input/mouse1"

A lot more info about the drivers and QWS_MOUSE_PROTO variable can be found on the following QT website.

Run the slider example: 

# cd /usr/share/qt/examples/widgets/sliders
# ./sliders -qws

IMPORTANT: Now you have configured your mouse manual, this can also be done automatically by using this export command: 

# export QWS_MOUSE_PROTO="Auto"

Using the touchscreen and the mouse together

The configuration method is exactly the same, only the export QWS_MOUSE_PROTO command will contain several parameters. For example to use the auto detection of the mouse and the touchscreen together, type: 

# export QWS_MOUSE_PROTO="Tslib:/dev/input/event1 Auto"

Saving the variables into your system

When you save the variables into your system, you can directly start working after you booted. You just need to edit the nfs_root/etc/profile file. Open a terminal and type: 

$ sudo gedit ~/nfs_root/etc/profile

And at the bottom of the file write the exports you want, those are the ones I used (for touchscreen and mouse): 

export TSLIB_TSDEVICE=/dev/input/event1
export TSLIB_TSEVENTTYPE=INPUT
export TSLIB_CONFFILE=/etc/ts.conf
export TSLIB_CALIBFILE=/etc/pointercal
export QWS_MOUSE_PROTO="Tslib:/dev/input/event1 Auto"

REMEMBER: You always have to run ts_calibrate and edit the etc/profile file every time you place a new filesystem inside nfs_root!!
