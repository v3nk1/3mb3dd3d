Linux-2.6.32.2 kernel mini2440 Porting (10) --- Configure USB peripherals

Transplant environment (red bold word to modify the contents of the blue bold special attention to the content)

1, the host environment: VMare under CentOS 5.5, 1G memory.

2, the integrated development environment: Elipse IDE

3, the compiler-compiler environment: arm-linux-gcc v4.4.3, arm-none-linux-gnueabi-gcc v4.5.1.

4, the development board: mini2440, 2M NOR Flash, 128M nand flash.

U-boot version: u-boot-2009.08

Linux version: linux-2.6.32.2

7, reference articles:

Embedded Linux application development entirely manual, Wei Dongshan, ed.

The Mini2440 of Linux transplantation develop combat guide

[1] hci device drivers in the kernel configuration USB keyboard and mouse

Linux-2.6.32.2 kernel support for USB peripherals is quite rich, and already contains S3C2440 USBHost driver support, so we only need to configure your kernel, you, here are a variety of USB peripherals kernel configuration.

In the implementation of make menuconfig Linux-2.6.32.2 root directory, and then confirm the following options are selected

Device Drivers --->

[*] HID Devices --->

<*> USB Human Interface Device (full HID) support / / support USB keyboard and mouse hci

Tip: The configuration options corresponding to the kernel source directory: linux-2.6.32.2/drivers/hid/usbhid USB keyboard and barcode scanner principle is the same, so the code is the same.

[*] / Dev / hiddev raw HID device support / / access to the original raw HID device.

The above information is selected can see the following in the console startup information

......

usbcore: registered new interface driver hiddev
usbcore: registered new interface driver usbhid
usbhid: v2.6: USB HID core driver

......

LCD and USB keyboard as a terminal

The kernel has support for LCD and USB keyboard can use it as a console terminal to ensure that likelihood by previous article said in command increase in line "console = tty1 and modify inittab file, when connected to a USB keyboard and LCD on the LCD and see the following message" Please press Enter to activate this console. "

It should be noted, point selected using a USB keyboard and LCD as an independent after the input and output devices, serial control terminal will no longer work in the development of the commissioning phase, or not so good as.

[2] U disk driver in the kernel configuration.

USB flash drives use the SCSI command, so let's add SCSI support.

Device Drivers --->

SCSI device support --->

<*> SCSI device support / / To support USB disk, this selected.

[*] Legacy / proc / scsi / support / / in the / proc / scsi directory.

<*> SCSI disk support / / SCSI hard disk, U disk support, this selected.
[*] USB support --->

<*> Support for Host-side USB / / usb host controller

[*] USB device filesystem (DEPRECATED) / / some debugging information in the / proc file system

<*> OHCI HCD support / / OHCI host controller driver

<*> USB Mass Storage support / / USB storage device

U disk mobile memory used mostly FAT/FAT32 format, so we need to add the FAT32 file system support in the kernel configuration under the main menu, select the following menu items:

File systems --->

DOS / FAT / NT Filesystems --->

<*> MSDOS fs support / / add the FAT32 file system support

- * - Native language support ---> / / The following is the encoding support in English

<*> Codepage 437 (United States, Canada)

<*> Simplified Chinese charset (CP936, GB2312)

<*> Traditional Chinese charset (Big5)

<*> NLS ISO 8859-1 (Latin 1; Western European Languages)

<*> NLS UTF-8
